## Project Description
DearBUP is a specialized blogging and social networking platform designed exclusively for the Bicol University-Polangui community. This repository contains the full-stack development progress, starting from Phase 1 (Planning) through Phase 7 (Deployment). The system serves as a bridge between students, campus organizations, and administration, providing a hub for storytelling and institutional updates.

---

## Project Objectives
* **Centralize University Communications:** To provide a platform where students can track, join, and evaluate school events efficiently.
* **Safe Emotional Expression:** To provide a secure and supportive environment for students to express thoughts and feelings.
* **Digital Bridge:** To act as a digital connection between students, campus organizations, and university administration.
* **Service Feedback:** To allow the community to share honest concerns and feedback regarding university services.

---

## Key Features
* **Creative Expression:** Allows students to share confessions customized with Spotify tracks for added emotional depth.
* **CBO Management:** A dedicated portal for accredited campus organizations to manage profiles and post updates.
* **Centralized Campus Updates:** Features an activity tracker and a real-time news feed for community engagement.
* **Institutional Feedback:** A system for students to provide honest, sometimes anonymous, feedback regarding university services.

# The project demonstrates our understanding of:
- Full-Stack Architecture: Connecting a React frontend to a Node.js/Express backend and MongoDB database.
- RESTful API Design: Implementing complete CRUD (Create, Read, Update, Delete) operations.
- Database Modeling: Designing Mongoose schemas and relationships for users, posts, and organizations.
- UI/UX Design: Applying a specialized color-graded design and typography using Tailwind CSS.
- Group Collaboration: Managing a shared codebase via GitHub with specialized technical roles.
- System Documentation: Developing professional diagrams including ERD, DFD, and Use Case diagrams.

## Weekly Development Roadmap
### Phase 1 - Project Planning (Week 10)
Established the system architecture, feature lists, and database schemas. Finalized the "DearBUP" proposal and initialized GitHub structure.

### Phase 2 - Backend Setup (Week 12)
Server initialization using Node.js and Express; API scaffolding and MongoDB schema creation.

### Phase 3 - Frontend Design (Week 12)
Development of layouts using React and Tailwind CSS based on the Phase 1 identity.

### Phase 4 - Form Submission & Data Insertion (Week 13)
Creating frontend forms and integrating them with backend APIs for storage in MongoDB.

### Phase 5 - Data Retrieval & Display (Week 14)
Fetching and displaying MongoDB data using JavaScript and Tailwind UI components.

### Phase 6 - Update & Delete Functionality (Week 15)
Implementing PUT and DELETE operations; integrating modals for CRUD completion.

### Phase 7 - Project Polishing & Documentation (Week 16)
Testing, debugging, UI/UX enhancements, and preparation for final defense.
