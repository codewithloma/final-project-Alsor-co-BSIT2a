## Project Description
DearBUP is a specialized blogging and social networking platform designed exclusively for the Bicol University-Polangui community. This repository contains the full-stack development progress, starting from Phase 1 (Planning) through Phase 7 (Deployment). The system serves as a bridge between students, campus organizations, and administration, providing a hub for storytelling and institutional updates.

---

# The project demonstrates our understanding of:
- Full-Stack Architecture: Connecting a Bootstrap frontend to a Node.js/Express backend and MongoDB database.
- RESTful API Design: Implementing complete CRUD (Create, Read, Update, Delete) operations.
- Database Modeling: Designing Mongoose schemas and relationships for users, posts, and organizations.
- UI/UX Design: Applying a specialized color-graded design and typography.
- Group Collaboration: Managing a shared codebase via GitHub with specialized technical roles.
- System Documentation: Developing professional diagrams including ERD, DFD, and Use Case diagrams.

## Weekly Development Roadmap
### Phase 1 - Project Planning (Week 10)
Established the system architecture, feature lists, and database schemas. Finalized the "DearBUP" proposal and initialized the GitHub repository structure. Defined the color-graded brand identity.

### Phase 2 - Backend Setup (Week 12)
Server initialization using Node.js and Express; API scaffolding and MongoDB/Mongoose schema creation based on Phase 1 planning.

### Phase 3 - Frontend Design (Week 12)
Development of interface layouts using HTML, CSS, and Bootstrap based on the Phase 1 identity; home, about, and basic navigation setup with custom color gradients.

### Phase 4 - Form Submission & Data Insertion (Week 13)
Creating frontend forms and integrating them with backend APIs; validating and storing community letters and feedback in MongoDB.

### Phase 5 - Data Retrieval & Display (Week 14)
Fetching and displaying MongoDB data using JavaScript and Bootstrap UI; building basic dashboards and news feed listings.

### Phase 6 - Update & Delete Functionality (Week 15)
Implementing PUT and DELETE operations; integrating modals for editing posts or removing data; final CRUD completion.

### Phase 7 - Project Polishing & Documentation (Week 16)
Testing, debugging, UI/UX enhancements, deployment on Render, and preparation for the final defense.
